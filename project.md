## Project Overview
- Next.js 16 App Router starter scaffold generated by create-next-app.
- React 19 with TypeScript; styling via Tailwind CSS v4 (PostCSS plugin) using inline theme vars in `app/globals.css`.
- Minimal custom config: default Next.js config, strict TypeScript, and eslint-config-next (core web vitals + TS) with global ignores aligned to Next defaults.

## Scripts
- `dev`: `next dev`
- `build`: `next build`
- `start`: `next start`
- `lint`: `eslint`

## Dependencies
- Runtime: `next@16.0.8`, `react@19.2.1`, `react-dom@19.2.1`
- Dev: TypeScript 5, ESLint 9 + `eslint-config-next`, Tailwind CSS 4 (via `@tailwindcss/postcss`), React/Node type defs.

## Project Structure
- `app/`
  - `layout.tsx`: Root layout importing global styles and loading Geist fonts; wraps all routes.
  - `page.tsx`: Home page with CTA links and brand assets using `next/image`.
  - `globals.css`: Tailwind `@import "tailwindcss";` with CSS custom props for theme and dark-mode overrides; sets base body styling and Tailwind theme vars.
- `public/`: Default Next/Vercel SVG assets.
- Config: `next.config.ts` (placeholder options), `tsconfig.json` (strict, path alias `@/*` to repo root), `eslint.config.mjs` (Next core-web-vitals + TS, custom ignores), `postcss.config.mjs` (Tailwind plugin).
- Metadata: `bun.lock`, `next-env.d.ts`, `README.md`.

## Styling Notes (Tailwind v4)
- Uses new PostCSS plugin `@tailwindcss/postcss`; Tailwind directives live in `globals.css`.
- Theme variables declared via `@theme inline` with CSS custom props for background/foreground and font tokens (`--font-geist-*`).
- Dark mode driven by `prefers-color-scheme` media query updating `:root` color tokens.

## Linting
- ESLint extends Next core-web-vitals + TypeScript presets.
- Ignores: `.next/**`, `out/**`, `build/**`, `next-env.d.ts`.

## TypeScript
- Strict mode on; `moduleResolution` set to `bundler`; `jsx` is `react-jsx`.
- Includes `**/*.ts, **/*.tsx, **/*.mts` and Next-generated type dirs; excludes `node_modules`.

## Running & Building
- Dev server: `npm|yarn|pnpm|bun run dev` -> `http://localhost:3000`.
- Production: `npm run build` then `npm run start`.

## References
- Based on Next.js App Router conventions (see Next.js docs via Context7 `/vercel/next.js`).
